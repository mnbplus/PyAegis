# Security Policy

## Supported Versions

PyAegis primarily provides security updates for the current major release.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

Security is the core of PyAegis. We take all security vulnerabilities seriously.

If you discover a security vulnerability within PyAegis itself (e.g. command injection in the CLI parsing, AST evasion techniques, Regex DoS in rule parsing), please do **NOT** open a public issue.

Instead, please send an email to `security@pyaegis.dev` or use the GitHub Security Advisories feature to privately report the vulnerability.

We will endeavor to respond to your report within 48 hours. Upon triaging, we will coordinate a fix and an advisory disclosure timeline with you.

### What to include
- A description of the vulnerability and its impact.
- Steps to reproduce the vulnerability (a Proof of Concept is highly appreciated).
- Potential remediation vectors if you have them.

Thank you for helping keep the ecosystem secure!
